# MASTER PROMPT: Automated YouTube Thumbnail Generator
## For GoHighLevel Affiliate Content Pipeline

> **How to use this**: Your Claude Code pipeline injects the variables in `{{double_braces}}` from your video script output and SEO data sources. The LLM receives this entire prompt, selects the appropriate template, generates the image prompt, and calls the image generation API. The post-processing layer then adds text, logo, and badges programmatically.

## BRAND DESIGN SYSTEM

### Color Palette (GoHighLevel Official + Supplements)

```
PRIMARY BACKGROUND (60%):
  - Dark Navy:       #0B223F (primary)
  - Navy Gradient:   #0B223F → #152E50 (default background treatment)
  - Deep Blue-Purple: #1A1A3E → #2D2D5E (alternate for variety — use every 4th-5th thumbnail)

SECONDARY ELEMENTS (30%):
  - Dodger Blue:     #188AF5 (GHL brand blue — UI elements, icons, secondary graphics)
  - Soft White:      #F0F4F8 (dashboard/screen mock elements)
  - Cool Gray:       #64748B (supporting graphic elements)

ACCENT / HIGHLIGHT (10%):
  - Selective Yellow: #FEB902 (GHL brand yellow — arrows, highlights, key numbers, badges)
  - Urgency Red:     #FF1744 (warning badges, "STOP" indicators, mistake thumbnails)
  - Success Green:   #00E676 (upward arrows, checkmarks, result indicators)
```

### Color Application Rules

- **60-30-10 distribution is mandatory.** 60% dominant background, 30% secondary elements, 10% accent. This ratio creates visual hierarchy that reads instantly at thumbnail scale.
- **Never use pure white (#FFFFFF) or pure black (#000000) as the dominant background.** YouTube's interface uses white (light mode) and #181818 (dark mode) — your thumbnail must have clear visual separation from both.
- **Accent colors are for focal points only.** Yellow arrow pointing to one thing = powerful. Yellow everywhere = noise.
- **Match accent to energy level:**
  - Yellow (#FEB902): tutorials, features, positive results, "how to" content
  - Red (#FF1744): mistakes, warnings, "stop doing this," urgency content
  - Green (#00E676): results, proof, money, growth, "what happened" content

### Typography Specifications (Post-Processing Layer)

```
HEADLINE TEXT:
  Font: Poppins Black (900 weight)
  Size: 90-120px depending on word count
  Color: #FFFFFF
  Stroke: 4px #000000 (ensures readability against ANY background)
  Position: Right third, vertically centered
  Max words: 4

EMPHASIS TEXT (numbers, results):
  Font: Poppins Black (900 weight)
  Size: 130-160px
  Color: #FEB902 (yellow) or #00E676 (green for money)
  Stroke: 5px #000000
  Position: Center or left-center depending on template

BADGE TEXT:
  Font: Poppins Bold (700 weight)
  Size: 36-44px
  Color: #FFFFFF
  Background: #FF1744 (red) or #FEB902 (yellow)
  Position: Top-left, rotated -15° diagonal banner
  Content: "2026" or "FREE" or "NEW" or "STOP"

LOGO:
  Asset: GoHighLevel logo (white version on dark backgrounds)
  Position: Top-left corner, below badge if badge present
  Size: 8% of frame width (~102px wide)
  Opacity: 90%
```

### Safe Zones (Non-Negotiable)

```
BOTTOM-RIGHT 15%: YouTube timestamp overlay — NEVER place critical elements here
BOTTOM 20px: Progress bar coverage — keep clear
TOP-RIGHT 10%: Desktop hover buttons — avoid primary elements
ACTION ZONE: Center 60-70% of frame — where eyes go first on mobile

TEXT OVERLAY ZONE: Right third of frame (427px wide) — kept as clean negative
space in the AI-generated image so programmatic text has a clear landing area

LOGO ZONE: Top-left corner (150x80px area)
```

## THE 5-TEMPLATE SYSTEM

### Template Selection Logic

```
IF CONTENT_FORMAT == "tutorial" OR "FAQ":
    → USE Template 1: "The Tutorial"

IF CONTENT_FORMAT == "quick-win" OR HOOK_TYPE == "proof":
    → USE Template 2: "The Result"
```

## PROMPT ASSEMBLY RULES

### Step 0: Pre-Generation — Competitive Pattern Analysis (Optional but Recommended)

If your pipeline has access to YouTube Data API or a scraping layer, run this BEFORE generating:

```
Analyze the thumbnails of the top 5 ranking YouTube videos for "{{target_keyword}}".
For each thumbnail, extract:
- Dominant color scheme
- Text hook (what words appear on the thumbnail)
- Layout pattern (face-left-text-right, centered, split, etc.)
- Primary visual element (face, screenshot, icon, number, object)

Output a summary of the dominant pattern and ONE specific way to visually
differentiate from it while remaining relevant to the topic.

Inject this into COMPETING_THUMBNAIL_ANALYSIS for the generation step.
```

### Step 1: Template Selection

Based on CONTENT_FORMAT and HOOK_TYPE, select the template using the logic above. If COMPETING_THUMBNAIL_ANALYSIS shows that all top videos use a similar template pattern, consider selecting an adjacent template instead to stand out.

### Step 2: Moment Translation

DO NOT describe the topic. Describe the VISUAL MOMENT of the transformation arc.

```
BAD (topic description):
"A thumbnail about GoHighLevel workflow automation"

BAD (feature description):
"A thumbnail showing the GoHighLevel Workflow Builder interface"

GOOD (visual moment):
"A stylized SaaS dashboard showing a dramatic automation flow with
glowing connection lines between trigger and action nodes, the
central workflow node illuminated in golden yellow suggesting the
'aha moment' of automation clicking into place"

GOOD (result moment):
"A glowing analytics chart with a sharp upward trend line breaking
through a ceiling barrier, golden particles exploding at the breakthrough
point, suggesting a dramatic performance improvement"
```

The moment should map to the Transformation Arc:
- **Nightmare-based hooks** → Show the PROBLEM visually (cluttered, red-tinted, complex)
- **Proof-based hooks** → Show the RESULT visually (clean, green/yellow, upward)
- **Contrarian hooks** → Show the DISCOVERY visually (spotlight, hidden, revealed)
- **Tutorial hooks** → Show the TOOL visually (clean interface, arrow pointing to feature)

### Step 3: Inject Variables into Template

Take the selected template's IMAGE GENERATION PROMPT STRUCTURE and replace:
- `[SPECIFIC UI ELEMENT related to {{ghl_feature}}]` with a visual description appropriate to the GHL feature (e.g., for "Workflow Builder" → "an automation flow diagram with connected nodes and trigger icons"; for "Reputation Management" → "a star rating display and review cards"; for "Chat Widget" → "a chat conversation interface with message bubbles")
- `[COMPETING TOOL's visual identity]` for comparison templates with a generic description of the competitor's aesthetic
- Any bracketed placeholders with video-specific visual descriptions

### Step 4: Append Global Negative Prompts

Add to EVERY prompt, regardless of template:

```
NEGATIVE: Do not include any text, words, letters, numbers, titles, subtitles,
watermarks, logos, brand names, URLs, or written content of any kind anywhere in
the image. Do not include human faces, hands, or body parts. Do not include
photography or photorealistic humans. Keep the composition clean and uncluttered
with maximum 3-4 distinct visual elements. Do not use busy patterns, grunge
textures, vintage effects, or low-contrast color schemes. Do not place any
important elements in the bottom-right 15% of the frame.
```

## POST-PROCESSING PIPELINE (Programmatic — runs after API returns the base image)

```python
# Pseudocode for the post-processing layer
# Use Pillow (PIL) or ImageMagick for implementation

def process_thumbnail(base_image, template_type, variables):
    """
    Takes the AI-generated base image and adds all text, logo, and badge
    elements programmatically for pixel-perfect typography control.
    """

    # 1. RESIZE to exact YouTube spec
    image = resize_and_crop(base_image, width=1280, height=720)

    # 2. ADD TEXT OVERLAY based on template
    #    - Font: Poppins Black (download from Google Fonts, include in pipeline assets)
    #    - Always apply black stroke/outline (4-5px) for readability
    #    - Position per template specification
    #    - Maximum 4 words for headline
    headline_text = variables['thumbnail_text']  # From video script metadata
    draw_text_with_stroke(
        image,
        text=headline_text.upper(),  # ALL CAPS for thumbnails
        font="Poppins-Black.ttf",
        size=calculate_font_size(headline_text),  # 90-120px based on word count
        color="#FFFFFF",
        stroke_color="#000000",
        stroke_width=4,
        position=get_text_position(template_type)  # Right third for most templates
    )

    # 3. ADD EMPHASIS NUMBER (Template 2 "The Result" only)
    if template_type == "result" and variables.get('quick_win_number'):
        draw_text_with_stroke(
            image,
            text=variables['quick_win_number'],
            font="Poppins-Black.ttf",
            size=150,
            color="#FEB902",  # GHL Yellow
            stroke_color="#000000",
            stroke_width=5,
            position="center_right_upper"
        )

    # 4. ADD BADGE (top-left diagonal banner)
    badge_text = select_badge(variables)  # "2026" | "FREE" | "NEW" | "STOP" | "WARNING"
    if badge_text:
        draw_badge(
            image,
            text=badge_text,
            bg_color="#FF1744",  # Red
            text_color="#FFFFFF",
            position="top_left",
            rotation=-15
        )

    # 5. ADD GHL LOGO (white version, top-left, below badge)
    logo = load_image("gohighlevel_logo_white.png")
    logo = resize(logo, width=int(1280 * 0.08))  # 8% of frame width
    paste_with_alpha(image, logo, position=(30, badge_height + 15))

    # 6. VALIDATE
    assert image.size == (1280, 720)
    assert get_filesize(image) < 2_000_000  # Under 2MB for YouTube
    assert passes_squint_test(image, scale=0.1)  # Readable at 120px wide

    # 7. GENERATE VARIATIONS
    # For A/B testing, generate 3 versions:
    #   Variation A: Default template output (above)
    #   Variation B: Swap accent color (yellow ↔ red)
    #   Variation C: Alternate text position (center instead of right third)

    return [variation_a, variation_b, variation_c]
```

```
2 words: 130px (high impact, rare)
1 word: 140-160px (use for single power words like "STOP" or "FREE")
5+ words: DO NOT USE. Go back and cut the thumbnail text to 4 words max.
```

## VARIATION STRATEGY FOR A/B TESTING

Generate 3 thumbnail variations per video for YouTube's Test & Compare feature:

```
VARIATION A — "Result-Focused"
  Primary visual: The outcome/result/metric visualization
  Text: Number or result-oriented hook
  Best when: Video has a concrete, quantifiable outcome

VARIATION B — "Feature-Focused"
  Primary visual: The GHL interface/tool/feature
  Text: Action-oriented hook ("Setup" "Build" "Create")
  Best when: Video is a walkthrough or tutorial

VARIATION C — "Text-Driven"
  Primary visual: Minimal — bold background with subtle graphic elements
  Text: Maximum impact text, larger font (130px+), fewer words (2-3)
  Best when: The hook words alone create enough curiosity
```

Let YouTube's algorithm determine the winner based on **watch-time share** (not just CTR — a thumbnail that attracts the wrong audience will have high CTR but low retention).
